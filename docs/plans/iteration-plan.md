# Sogon 이터레이션 플랜

> MVP 완성 이후, 새 기능 추가를 통한 디자인 iteration 계획.
> 디자이너 협업이 필요한 기능과 독립 개발 가능한 AI 기능으로 분리.

---

## Part 1: 디자이너 협업 기능

디자인 → 구현 → iteration 사이클을 함께 돌릴 기능들. 순서는 미정.

### 1. 설정 페이지

현재 네비게이션에 disabled 버튼만 있고 페이지가 없음.

**기능 범위**
- 비밀친구 톤 변경 (온보딩 선택 이후에도 변경 가능)
- 계정 정보 확인 (이메일, 가입일)
- 로그아웃 (현재 모바일에서 불가)
- 테마 설정 (다크 모드와 연결)

**디자이너와 할 일**
- 설정 페이지 레이아웃 설계
- 섹션 구성과 계층 구조

---

### 2. 다크 모드

소곤의 따뜻한 브랜드에 맞는 "warm dark" 톤 테마.

**기능 범위**
- 3가지 옵션: 브라우저 설정 따라가기 / 다크 / 화이트
- 기본값: 설정이 없으면 화이트
- 설정 저장: localStorage (DB 저장 불필요)

**디자이너와 할 일**
- 다크 모드 컬러 팔레트 설계 (순수 검정 X, 차콜/다크그린 계열)
- 테마 토글 UI 컴포넌트

---

### 3. 감정/무드 태깅

기록할 때 그날의 기분을 이모지로 태깅. 선택은 optional.

**기능 범위**
- 5가지 무드 이모지 (😊 좋음 / 😌 평온 / 😢 슬픔 / 😤 짜증 / 😴 피곤)
- 기록 작성 시 무드 피커 UI
- 기록 카드에 무드 이모지 배지 표시
- 캘린더 뷰에서 날짜 셀에 그날의 대표 무드 이모지 표시

**디자이너와 할 일**
- 무드 이모지 세트 결정 (이모지 or 커스텀 아이콘)
- 무드 피커 인터랙션 디자인
- 캘린더 날짜 셀 디자인 변경

---

### 4. 무드 통계 대시보드

무드 태깅 데이터 기반 시각화. 디자이너와 가장 크리에이티브하게 협업할 영역.

> **의존성**: 무드 태깅(#3) 구현 필요

**기능 범위**
- 무드 트렌드: 최근 30일 무드 변화 차트
- 무드 분포: 전체 기간 무드별 비율 도넛/바 차트
- 기록 빈도 히트맵 (GitHub contribution 그래프 스타일)
- 연속 기록일 스트릭: 현재 + 최장 기록
- 이번 달 하이라이트: 가장 많이 기록한 날, 가장 긴 기록 등

**디자이너와 할 일**
- 대시보드 레이아웃 전체 설계
- 차트 타입과 스타일 결정
- 무드별 컬러 매핑
- 차트 등장 애니메이션

---

### 5. 비밀친구 커스터마이징

비밀친구에게 이름을 짓고 프로필을 꾸밀 수 있는 기능.

**기능 범위**
- 비밀친구 이름 짓기 (기본값: "비밀친구")
- 프로필 이모지 또는 간단한 아바타 선택
- 리액션 버블에 커스텀 이름/아바타 반영

**디자이너와 할 일**
- 아바타 선택 UI
- 캐릭터 에디터 경험 설계
- 커스텀 이름이 적용된 리액션 버블 디자인

---

### 6. 주간 요약 리포트

한 주간의 기록을 AI가 돌아보고 요약해주는 새 뷰.

**기능 범위**
- 주간 선택기 (이번 주 / 지난 주 / 2주 전 ...)
- AI 요약 카드 (비밀친구 톤으로 작성)
- 이번 주 하이라이트 자동 선별 (가장 긴 기록, 사진 포함 기록 등)
- 무드 요약 — 무드 태깅 완료 시 연동

**디자이너와 할 일**
- 요약 페이지 레이아웃
- 하이라이트 카드 디자인
- 기록이 없는 주의 빈 상태 디자인

---

### 7. 검색/필터

기록이 쌓인 후 과거 기록을 찾을 수 있는 기능.

**기능 범위**
- 컬렉션 페이지 상단 검색바
- 실시간 검색 + 검색어 하이라이트
- 빈 결과 상태

**디자이너와 할 일**
- 검색바 UI 디자인
- 검색 결과 표시 방식
- 빈 결과 상태 디자인

---

## Part 2: 독립 개발 기능 (AI 중심)

디자인 작업 없이 혼자 진행할 수 있는 AI/백엔드 기능.

### A. AI 시스템 프롬프트 구조화

현재 `generate-reaction.ts`에 하드코딩된 시스템 프롬프트를 별도 마크다운 파일로 분리. 코드 수정 없이 프롬프트만 편집 가능하도록.

### B. AI 멀티모달 리액션

사진이 첨부된 기록 시 이미지도 LLM에 함께 전달. 이미지 내용을 인지하고 반응할 수 있도록 구조 개선.

### C. AI 메모리 (이전 기록 반영)

리액션 생성 시 최근 기록을 컨텍스트로 전달하여 반복적/지루한 리액션 방지.

### D. 기록 리마인더

"오늘 아직 안 썼는데, 어땠어?" 같은 리마인더. 브라우저 Push Notification 또는 앱 내 배너.

## 후보 비교

### 디자이너 협업 기능

| 기능 | 디자인 작업량 | 디자이너 협업 가치 | 의존성 |
|------|:-----------:|:----------------:|:------:|
| 설정 페이지 | 중간 | 중간 | 없음 |
| 다크 모드 | 높음 | 높음 | 설정 페이지와 묶으면 좋음 |
| 감정/무드 태깅 | 높음 | 높음 | 없음 |
| 무드 통계 대시보드 | 매우 높음 | 매우 높음 | 무드 태깅 필요 |
| 비밀친구 커스터마이징 | 높음 | 높음 | 없음 |
| 주간 요약 리포트 | 높음 | 높음 | 없음 |
| 검색/필터 | 중간 | 중간 | 없음 |

### 독립 개발 기능

| 기능 | 난이도 | 사용자 임팩트 |
|------|:------:|:-----------:|
| 프롬프트 구조화 | 낮음 | 간접적 |
| AI 멀티모달 | 중간 | 높음 |
| AI 메모리 | 중간 | 높음 |
| 기록 리마인더 | 중간 | 중간 |
