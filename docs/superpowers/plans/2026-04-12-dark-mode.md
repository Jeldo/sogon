# Dark Mode UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the entire UI respond to dark mode by replacing hardcoded colors with semantic CSS variable tokens.

**Architecture:** Define semantic color tokens (background, surface, elevated, border, text-*) in globals.css with light/dark values. Replace all hardcoded Tailwind color classes across 12 files with the semantic token classes. The existing theme toggle and `.dark`/`.system-theme` class system already works тАФ this plan only adds the missing color overrides and updates component classes.

**Tech Stack:** Tailwind CSS v4, CSS custom properties, Next.js 16
