# Sogon MVP Wireframe Design Spec

## Context

소곤(Sogon)은 프라이빗한 일기장에 AI 비밀친구가 리액션해주는 서비스다. PRD v1.0 기반 MVP 4개 화면의 와이어프레임을 정의한다. 디자인 가이드(`DESIGN_GUIDE.md`)의 컬러, 타이포그래피, 스페이싱, 컴포넌트 토큰을 따른다.

---

## 핵심 결정 사항

| 결정 | 내용 |
|------|------|
| 기록 ↔ 리액션 관계 | **분리형** — 기록 제출 후 리액션 화면으로 자동 전환 |
| 모아보기 기본 뷰 | **캘린더** — 타임라인은 탭 전환 |
| 온보딩 포함 여부 | 포함 (비밀친구 톤 선택) |

---

## 전체 네비게이션 구조

### 데스크톱 (>= 1024px)
- **좌측 사이드바** (240px, 접힘 시 64px)
  - 로고 (소곤)
  - 기록하기 (pen-line 아이콘)
  - 모아보기 (book-open 아이콘)
  - 설정 (settings 아이콘, 하단 고정)
- **중앙 콘텐츠** — 화면별 max-width 적용

### 모바일 (< 768px)
- **하단 탭 내비게이션**
  - 기록하기 / 모아보기 / 설정

### 유저 플로우
```
첫 진입 → 온보딩 (비밀친구 선택) → 기록하기 (메인)
기록 제출 → 리액션 화면 (자동 전환) → "하나 더 기록하기" 또는 "모아보기"
모아보기 → 캘린더(기본) / 타임라인(탭)
```

---

## 1. 온보딩 — 비밀친구 선택

**레이아웃**: 리스트 카드형, 560px 중앙 정렬

### 구성 요소
- **상단**: 로고 아이콘 + "소곤" 텍스트 + "비밀친구를 골라봐!" 서브텍스트
- **중앙**: 세로로 3개 카드 나열
  - 각 카드: 이모지 아이콘 (40x40, 컬러 배경 원형) + 톤 이름 + 예시 리액션
  - 카드 스타일: `bg-white`, `border 2px solid neutral-200`, `rounded-xl (16px)`, `padding 16px`
  - 선택 시: `border-color: primary-500`, `box-shadow: 0 0 0 3px rgba(110, 189, 90, 0.15)`
- **하단**: "시작하기" Primary Button

### 톤 카드 데이터

| 톤 | 이모지 | 배경색 | 예시 리액션 |
|----|--------|--------|------------|
| 따뜻한 친구 | 🤗 | #fde8e8 | "그랬구나, 고생했다 오늘도" |
| 쿨한 친구 | 😎 | #e0f0ff | "오 괜찮은데? 잘했네" |
| 텐션 높은 친구 | 🎉 | #fff8e0 | "헐 대박!! 완전 좋다!!" |

### 인터랙션
- 카드 hover: `scale: 1.02`, 200ms
- 카드 선택: 글로우 효과 + 체크 표시
- "시작하기" 버튼: 톤 선택 전에는 비활성화

## 2. 기록하기 — 메인 입력 화면

**레이아웃**: 페이지형 + 오늘 기록 미리보기, 720px max-width

### 구성 요소
- **사이드바** (240px, 펼침 상태)
  - 로고 + "소곤"
  - 네비 항목: 기록하기(활성), 모아보기, 설정
- **메인 콘텐츠**
  - **헤더**: "기록하기" 타이틀 + 날짜 (예: "4월 5일 토요일")
  - **입력 영역**:
    - Textarea: `min-height 120px`, `rounded-lg (16px)`, `padding 20px`
    - Placeholder: "오늘 뭐 했어?"
    - 폰트: Gowun Dodum, text-lg (18px)
  - **액션 바**: 좌측에 사진 첨부 버튼 (image-plus 아이콘), 우측에 "기록하기" Primary Button (send/arrow-up 아이콘)
  - **오늘의 기록 미리보기** (구분선 아래):
    - "오늘의 기록" 라벨
    - 오늘 작성한 기록 카드 리스트 (최신순)
    - 각 카드: 기록 텍스트 + 비밀친구 리액션 (Nanum Pen Script, 이탤릭) + 타임스탬프

### 인터랙션
- Textarea focus: `border-color: primary-500`, `focus shadow`
- "기록하기" 버튼: 텍스트 비어있으면 비활성화
- 사진 첨부: 클릭 시 파일 선택, 첨부 후 textarea 하단에 썸네일 미리보기
- 제출 후: 리액션 화면으로 자동 전환

## 3. 리액션 — 비밀친구 반응 화면

**레이아웃**: 카드 대화형, 480px max-width 중앙 정렬

### 구성 요소
- **사이드바**: 아이콘 전용 모드 (64px)
- **메인 콘텐츠** (중앙 정렬)
  - **내 기록 카드**:
    - 라벨: "방금 나의 기록" (text-xs, neutral-400)
    - 카드: `bg-white`, `border neutral-200`, `rounded-xl (20px)`, `padding 20px`
    - 텍스트: 방금 작성한 기록 내용
  - **비밀친구 리액션 카드** (아래에 등장):
    - 카드: `bg-neutral-100`, `rounded-xl (20px)`, `padding 20px`
    - 아바타: 32px 원형 아이콘 + "비밀친구" 라벨 (primary-600 색상)
    - 리액션 텍스트: Nanum Pen Script, text-xl (20px)
  - **액션 버튼** (하단, 가로 나란히):
    - "하나 더 기록하기" — Secondary Button → 기록하기 화면으로
    - "모아보기" — Ghost Button → 모아보기 화면으로

### 리액션 등장 애니메이션 (시그니처 모먼트)
1. **타이핑 인디케이터**: 세 점 펄스 (opacity 0.3→1→0.3, 1.2초 루프)
2. **1~2초 딜레이** 후 리액션 버블 등장
3. **버블 등장**: `opacity 0→1`, `translateY 8px→0`, `scale 0.97→1.0`
4. Duration: 500ms, easing: `ease-spring`

## 4. 모아보기 — 캘린더(기본) + 타임라인

**레이아웃**: 캘린더 위 + 기록 리스트 아래 (1컬럼), 840px max-width

### 구성 요소
- **사이드바** (240px, 펼침 상태)
  - 모아보기 활성 상태
- **메인 콘텐츠**
  - **헤더**: "모아보기" 타이틀 + 뷰 전환 탭 (캘린더 / 타임라인)
    - 탭 스타일: pill 형태, `bg-neutral-100`, 활성 탭은 `bg-white + shadow-sm`
  - **캘린더 영역**:
    - `bg-white`, `border neutral-200`, `rounded-lg (16px)`, `padding 20px`
    - 월 네비게이션: 좌우 화살표 + "2026년 4월"
    - 요일 헤더: 일~토
    - 날짜 그리드: 7열
      - 기록 있는 날: 하단에 `primary-400` (#93D17E) 도트
      - 오늘: `ring border primary-300`
      - 선택된 날: `bg-primary-600`, 흰 텍스트, `rounded-full`
    - 날짜 셀: 40x40px, `rounded-full`
  - **선택 날짜 기록 리스트** (캘린더 아래):
    - 날짜 라벨: "4월 5일 토요일"
    - 기록 카드 리스트 (시간순):
      - `bg-white`, `border neutral-200`, `rounded-xl (16px)`, `padding 16px`
      - 기록 텍스트 (text-base, Gowun Dodum)
      - 비밀친구 리액션 (Nanum Pen Script, text-xl, neutral-500)
      - 타임스탬프 (text-xs, neutral-400)

### 타임라인 뷰 (탭 전환 시)
- 최신순 기록 카드 스크롤
- 카드 구성은 위와 동일
- 날짜 구분선으로 그룹핑 (예: "4월 5일 토요일", "4월 4일 금요일")
- 스크롤 진입 시 stagger 등장 애니메이션: `translateY 12px→0`, `opacity 0→1`, 350ms, stagger 50ms

## 빈 상태 (Empty State)

| 화면 | 조건 | 표시 내용 |
|------|------|----------|
| 기록하기 — 오늘 기록 미리보기 | 오늘 기록 없음 | 미리보기 영역 자체를 숨김. 입력 영역만 표시 |
| 모아보기 — 캘린더 날짜 선택 | 해당 날짜 기록 없음 | "이 날은 기록이 없어. 오늘 하나 남겨볼까?" + "기록하기" 링크 |
| 모아보기 — 타임라인 | 전체 기록 없음 | "아직 기록이 없어. 첫 기록을 남겨봐!" + "기록하기" 버튼 |
