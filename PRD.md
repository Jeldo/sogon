# 소곤 (Sogon) PRD v1.0

## 1. 개요

### 서비스명
소곤 — "소곤소곤"에서 따온 이름. 조용히, 나만의 공간에서 속삭이듯 하루를 기록하는 서비스.

### 한 줄 요약
**프라이빗한데 반응이 있는 하루 기록 공간** — 비밀친구가 내 하루에 리액션해주는 일기장.

### 배경 및 문제 정의
사람들은 하루에도 여러 번 "이거 누군가에게 말하고 싶다"는 순간을 겪는다. 하지만:
- **SNS**: 타인의 시선이 부담스럽다. 좋아요 수, 반응에 신경 쓰게 된다.
- **메신저**: 누군가에게 따로 연락하기엔 내용이 애매하다. 부담을 줄까 걱정된다.
- **일기장**: 쓰는 건 좋지만, 아무런 반응이 없어 외롭다.

**핵심 인사이트**: 가감 없이 나를 표현하면서도 리액션을 받을 수 있는 공간이 없다.

---

## 2. 타겟 유저

### 1차 타겟
- SNS에 글을 올리기 부담스러운 사람
- 일상의 작은 순간을 누군가와 나누고 싶지만, 특정 상대에게 연락하기엔 애매한 사람
- 혼자 보내는 시간이 많지만 고립감을 느끼고 싶지 않은 사람

### 유저 시나리오
> 퇴근길, 하늘이 예뻐서 사진을 찍었다. 인스타에 올리자니 너무 사소하고, 친구에게 보내자니 "갑자기 이걸 왜?" 싶다. 소곤을 열고 사진과 함께 "오늘 하늘 진짜 예쁘다"라고 적는다. 몇 초 뒤, 비밀친구가 "우와 진짜 예쁘다! 이런 하늘 보면 괜히 기분 좋아지지 않아?" 라고 반응해준다.

## 3. 핵심 가치

| 키워드 | 설명 |
|--------|------|
| **프라이빗** | 완전히 나만의 공간. 누구에게도 공개되지 않는다. |
| **부담 없음** | 짧든 길든, 사소하든 진지하든, 뭘 써도 괜찮다. |
| **반응이 있음** | 비밀친구가 항상 내 글에 리액션해준다. 혼자가 아닌 느낌. |

## 4. MVP 기능 명세

### 4.1 온보딩 — 비밀친구 선택

**설명**: 서비스 첫 진입 시 비밀친구의 성격(톤)을 선택한다.

- 인증 없이 기기 기반으로 시작 (별도 회원가입/로그인 없음)
- 2~3가지 톤 선택지 제공:

| 톤 | 성격 | 리액션 예시 |
|----|------|------------|
| **따뜻한 친구** | 공감 잘 해주고, 다정하게 반응 | "그랬구나, 고생했다 오늘도" |
| **쿨한 친구** | 담백하고 솔직하게, 가볍게 반응 | "오 괜찮은데? 잘했네" |
| **텐션 높은 친구** | 에너지 넘치고 리액션이 큰 편 | "헐 대박!! 완전 좋다!!" |

- 선택 후에도 설정에서 변경 가능

### 4.2 하루 기록 (핵심 기능)

**설명**: 부담 없이 짧게 하루의 순간을 기록한다.

- **입력 형태**: 텍스트 + 사진 (선택)
  - 텍스트: 글자 수 제한 없음 (단, UI는 짧은 글에 최적화)
  - 사진: 1장 첨부 가능
- **기록 빈도**: 하루 횟수 제한 없음
- **UI 방향**:
  - 앱 진입 시 바로 입력 화면
  - 메신저에 메시지를 보내는 듯한 간결한 UX
  - "오늘 뭐 했어?" 같은 가벼운 플레이스홀더로 진입 장벽 낮추기

### 4.3 비밀친구 리액션 (핵심 기능)

**설명**: 유저가 글을 쓰면, 비밀친구가 친구처럼 한마디 반응해준다.

- **리액션 성격**:
  - 분석 리포트 X, AI 상담사 X
  - **친구처럼** 자연스러운 한마디
  - 유저가 선택한 톤에 맞는 말투
- **리액션 타이밍**: 기록 제출 직후 (실시간)
- **리액션 길이**: 1~2문장 (너무 길지 않게)
- **리액션 원칙**:
  - 좋은 일 → 같이 기뻐해주기
  - 힘든 일 → 공감해주기
  - 사소한 일 → 관심 가져주기
  - 사진이 있으면 → 사진에 대해서도 반응

### 4.4 기록 모아보기

**설명**: 지난 기록과 비밀친구의 반응을 돌아볼 수 있다.

- **뷰 모드 2가지** (탭 전환):
  - **타임라인**: 최신순으로 내 글 + 비밀친구 리액션을 스크롤하며 보기
  - **캘린더**: 월간 달력에서 기록이 있는 날짜 표시 → 날짜 탭하면 해당 날의 기록 목록
- 각 기록 카드에 표시되는 정보:
  - 작성 시간
  - 텍스트 내용
  - 사진 (있는 경우)
  - 비밀친구 리액션

## 5. MVP에서 제외하는 기능 (향후 로드맵)

| 기능 | 우선순위 | 비고 |
|------|----------|------|
| 회원가입/로그인 (Supabase Auth) | 높음 | MVP 이후 바로 추가. 기기 변경/데이터 보존 위해 |
| 주간 요약 리포트 | 중간 | claude-sonnet 활용, 한 주 돌아보기 |
| 검색/필터 | 중간 | 기록이 쌓인 후 필요 |
| 비밀친구 캐릭터 커스터마이징 | 낮음 | 이름 짓기, 프로필 이미지 등 |
| 알림/리마인더 | 낮음 | "오늘 아직 안 썼는데, 어땠어?" |

## 6. 핵심 유저 플로우

```
[첫 진입]
  → 비밀친구 톤 선택 (따뜻한 / 쿨한 / 텐션 높은)
  → 메인 화면 (기록 입력)

[기록 작성]
  → 텍스트 입력 (+ 사진 선택)
  → 전송
  → 비밀친구 리액션 표시
  → 계속 기록하거나 모아보기로 이동

[기록 모아보기]
  → 타임라인 / 캘린더 탭 전환
  → 과거 기록 + 리액션 열람
```

## 7. 리스크 및 고려사항

| 리스크 | 대응 방안 |
|--------|----------|
| AI 리액션이 반복적/지루해질 수 있음 | 다양한 반응 패턴 설계, 맥락(이전 기록) 반영 |
| 민감한 내용 (자해, 위기 상황) 입력 시 | 위기 상황 감지 시 전문 상담 연결 안내 표시 |
| 인증 없이 시작 → 데이터 유실 리스크 | 앱 내에서 "계정 연결하면 데이터 보존" 유도 (후속) |
| 사진 저장 비용 증가 | 이미지 압축 + 저장 용량 모니터링 |
